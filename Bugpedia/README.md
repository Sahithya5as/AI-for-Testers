# 🐞 Bugpedia

Bugpedia is a structured bug documentation and QA workflow framework designed to standardize bug reporting and improve communication between testers and developers.

Many teams struggle not because bugs exist, but because bugs are reported unclearly.  
Bugpedia solves that problem by providing a reproducible, structured reporting approach.

---

## Why I built this
During testing I noticed:
- Developers couldn’t reproduce issues
- Bugs were marked "Not Reproducible"
- Severity and priority were confused
- Communication gap between QA and Dev delayed releases

So I created a reusable bug reporting system.

---

## What Bugpedia Provides
• Standard bug report template  
• Reproducibility checklist  
• Severity vs Priority classification  
• Mobile + API testing scenarios  
• Fintech failure scenarios (timeouts, retries, bank downtime)  
• Negative testing guidance  
• Edge case identification
